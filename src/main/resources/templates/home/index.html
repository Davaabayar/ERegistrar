Implementing a new Spring Web Application - eRegistrar (30 points)

Create a new springboot-springwebmvc webapp named, eRegistrar, which is an online Registration system for a University, which you should implement as follows:

Using Spring Boot, implement a simple enterprise web application named eRegistrar. Your implementation should make use of the following tools, technologies, frameworks and APIs: Spring Web MVC, Thymeleaf, JPA and a relational database. You are expected to implement the following 4 basic use-cases:

  1. Display a homepage for the web app (e.g. url - http://localhost/eregistrar/home etc) which presents menu items/links on a navigation bar.
  2. Display List of Students - presents to the user, a list of all the students who have been registered in the university. You may display the data in an html grid or table or an ordered list.
  3. Register a New Student - enables the user (i.e. the university registrar) to add a new Student into the system, using a web form UI.
  4. Edit a Student's registration data.
  5. Delete a student's registration data.
Note:
The specification for Student is as follows:
  • studentId : (should be primary key field)
  • studentNumber: e.g. 000-61-0001 (required)
  • FirstName: e.g. Anna (required)
  • MiddleName: (optional)
  • LastName: e.g. Smith (required)
  • cgpa: e.g. 3.78 (optional)
  • enrollmentDate: e.g. 2019-5-12 (required)
  • isInternational: (values: "Yes", "No") (required) - Hint: Implement/present this data in a drop-down list

When complete:Take screenshots of your eRegistrar screens and add them to a subfolder inside your project named screenshots; zip your project folder into a zip file named, eregistrar.zip and upload/Submit here on Sakai or push to your github and submit its url to sakai. (Advice: Use the elibrary project as a guide)
